## 二、各功能模块说明

### 2.1 NLP分析模块

基于 **hfl/chinese-roberta-wwm-ext** 在自建心理健康领域对话数据集上进行多任务微调，实现三项分析任务：

| 任务 | 类别/输出 | 说明 |
|---|---|---|
| 细粒度情绪分类 | 7类：平静、焦虑、悲伤、愤怒、恐惧、羞耻、绝望 | 交叉熵损失 + 逆频率类别权重 |
| 用户意图识别 | 6类：情绪宣泄、寻求共情、寻求建议、自我披露、信息咨询、危机求助 | 与情绪分类共享编码器 |
| 情绪强度评估 | 连续值 [0, 1] | 回归头，MAE = 0.0923 |
| 风险辅助评分 | 连续值 [0, 1]（辅助特征） | 供 RiskService 综合判断使用 |

**与BERT基线对比（490条均衡测试集，每类情绪各70条）：**

| 指标 | RoBERTa（部署模型）| BERT（基线）|
|---|---|---|
| 情绪准确率 | **72.86%** | 68.57% |
| 情绪宏平均F1 | **72.55%** | 68.24% |
| 意图准确率 | **73.67%** | 73.47% |
| 强度 MAE | **0.0923** | 0.1023 |
| 危机探针（30题）| **30/30 ✅** | 28/30 ❌ |

> **说明：** 情绪准确率（72.86%）低于开题报告目标值（≥80%）。主要原因：（1）训练数据规模有限，为自建心理领域数据集；（2）7类细粒度分类任务本身难度较高，尤其是"羞耻"与"恐惧"的边界模糊；（3）均衡测试集相比人工标注集更严格（人工标注集因类别严重不均衡，准确率上限约为0.40）。危机探针30题全部通过，安全性要求满足。

### 2.2 Prompt工程与LLM集成

- **LLM接入：** 火山方舟（Doubao）API，OpenAI兼容接口，支持流式输出（SSE）
- **多轮上下文管理：** 三层结构
  - 第一层：原始数据库记录（全量历史）
  - 第二层：对话摘要（每4条新消息更新一次，超过10轮后启用）
  - 第三层：运行时窗口（最近6–10条消息 + 摘要，注入Prompt）
- **结构化NLP分析注入：** 每轮对话中，将以下内容作为 `system` 消息插入LLM消息链：
  ```
  [当前情绪分析]
  情绪类型：anxiety，意图类型：venting，情绪强度：60%，风险评分：0.18
  风险等级：L1，触发原因：无
  请根据以上分析调整回复策略：…
  ```
  使LLM能够针对不同意图（如情绪宣泄 vs. 寻求建议）和不同情绪强度采用差异化的支持策略。

> **关于LangChain：** 开题报告提及使用LangChain框架，实现中直接使用OpenAI SDK完成了等价的Prompt构建与多轮编排，未引入LangChain依赖，功能等价，工程上更轻量。

### 2.3 危机预警机制

采用**模型评分 + 规则关键词**双重触发机制：

| 风险等级 | 触发条件 | 系统行为 |
|---|---|---|
| L0 正常 | 无触发 | 标准回复 |
| L1 关注 | 模型风险评分 > 0.3 | 后台记录，对话继续 |
| L2 警告 | 关键词命中或评分 > 0.5 | 后台预警记录，提示心理援助资源 |
| L3 危机 | 危机关键词命中（自伤/自杀相关）| 前端展示危机干预卡片（含4条心理援助热线），LLM切换危机支持话术，后台记录紧急预警 |

已在数据库 `resource_catalog` 表预置4条危机援助热线（北京心理危机研究与干预中心、全国心理援助热线、生命热线、希望24热线）。

### 2.4 用户对话界面（前端）

- 匿名会话创建，`visitor_id` 持久化至 `localStorage`
- 历史会话选择器：页面刷新后可选择继续历史对话或开始新对话
- 流式SSE接收，打字机效果实时显示LLM回复
- 危机卡片（L3触发时显示援助热线）
- 对话界面不显示情绪分析标签（NLP数据仅对咨询师可见）

### 2.5 咨询师管理后台

访问受密码保护（密码：`xinyu2026`），会话数据与用户隐私不对外暴露。

**数据统计面板：**

## 三、系统架构概览

```
用户浏览器
    │  HTTP GET /               → 前端 React SPA（FastAPI StaticFiles）
    │  POST /api/v1/sessions    → 创建会话
    │  POST /api/v1/chat/{id}/stream → SSE 流式对话
    │
FastAPI 后端（uvicorn）
    ├── NLPService (RealNLPService)
    │       └── MentalHealthMultiTaskModel (RoBERTa-wwm-ext 微调)
    ├── RiskService
    │       └── 模型评分 + L2/L3关键词规则
    ├── ContextService
    │       └── 三层上下文窗口（摘要 + 最近消息）
    ├── PromptService
    │       └── 动态Prompt构建（含NLP结构化注入）
    ├── DoubaoLLMProvider
    │       └── 火山方舟 API（OpenAI兼容，流式输出）
    └── ResourceService
            └── 危机援助热线查询

PostgreSQL 16
    └── 8张业务表：visitor_profiles, chat_sessions, chat_messages,
        message_analyses, conversation_summaries, alert_events,
        resource_catalog, counselor_accounts
```

## 六、与开题报告要求的对照说明

| 开题报告指标 | 目标 | 实际达成 | 说明 |
|---|---|---|---|
| 情绪类别数 | ≥5种 | **7种** | 超出目标 |
| 情绪准确率 | ≥80% | **72.86%** | 略低于目标，见下方说明 |
| 意图识别 | 区分倾诉/求助/闲聊等 | **6类意图** | 满足要求 |
| 危机预警 | 前端提示 + 资源推送 + 后台记录 | **✅ 全部实现** | |
| 情绪可视化 | 图表呈现情绪变化趋势 | **✅ 咨询师后台详情页** | 用户侧不展示，仅咨询师可见 |
| 咨询师后台 | 会话统计 + 情绪分布分析 | **✅ 完整实现** | 含饼图、柱状图、预警管理 |
| 多轮对话 | 维护会话历史，保障连贯性 | **✅ 三层上下文** | |
| 安全约束 | 过滤不当内容 | **✅ L3危机安全模式** | LLM切换至危机话术，不进行开放讨论 |

**关于情绪准确率（72.86% vs. 目标80%）的说明：**

本系统采用7类细粒度情绪分类，分类难度高于常见的二/三类情感分析。训练数据为自建心理健康领域对话数据集，规模有限。在均衡测试集（每类各70条）上，RoBERTa-wwm-ext 准确率达72.86%，较BERT基线（68.57%）提升4.29个百分点。在安全性关键的危机识别任务上，系统达到30/30（100%），满足最严格的安全要求。论文评估部分将对此差距进行分析说明。
